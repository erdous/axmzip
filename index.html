// algorithm

How the axioms
compound

Each pass discovers structure at a higher level of abstraction. By the final pass, most data reduces to a handful of axiom IDs and a small library that generates everything.

// Pass 01

Atomic Discovery

Scan the binary stream at multiple window sizes. For each recurring pattern, calculate net savings — bytes saved minus library overhead. Register the top patterns as atomic axioms.

// Pass 02+

Compound Discovery

The stream is now a sequence of IDs. Find frequently co-occurring adjacent pairs. Each useful pair becomes a compound axiom referencing two existing ones. Sequence halves each pass.

// Final

Pruning & Serialization

Unused library entries are discarded. IDs remapped 0..N. In the best case, 193 candidates collapse to 7. Output: binary blob with varint IDs and compact axiom library.

This mirrors how biological memory works. You don't store every frame of a memory — you store the generating rules, and reconstruct on demand. Axmzip does the same thing with bits.

// usage

Three lines
to compress anything

Pure Python. Zero dependencies. MIT licensed.

example.py 
from axmzip import Axmzip

# Compress any bytes
ez   = Axmzip()
blob, stats = ez.compress(data)

# stats: {'original_bytes': 18000, 'compressed_bytes': 1062,
#         'ratio_pct': 94.1, 'lib_entries': 60, 'seq_len': 49}

# Decompress — always lossless, MD5 verified
recovered = ez.decompress(blob)
assert recovered == data  # guaranteed
